# 量潮沟通云交互设计文档

## 交互模式

### 概述

如果把这三个模式比作一个"信息工厂"：

- **对话模式是"原材料采集车间"**：工人（用户）把矿石（想法）挖出来，初步筛选成小块（便签）
- **白板模式是"加工组装车间"**：工程师（用户）把这些小块按照图纸（逻辑）拼装成一个个零件或半成品（备忘录）
- **画布模式是"包装发货车间"**：销售经理（用户）把半成品按照客户的要求（沟通对象）进行精美包装（润色），然后贴上快递单（发送/发布），把它送到客户手中

### 对话模式 (Dialogue Mode)

这是整个工作流的起点，模拟人最自然的思维方式——"自言自语"或"与AI对话"。

**定位**：灵感捕捉与信息入口
**核心交互**：聊（输入消息）
**用户行为**：像使用聊天软件或大模型官网一样，通过一问一答的形式，把脑子里模糊的想法、遇到的问题、零散的信息"倒"出来。
**AI角色**：一个聪明的倾听者和记录员，负责理解用户的意图，并将大段的对话内容切片。
**产出物**：便签。AI 会自动将对话中的关键信息、待办事项、灵感要点，提炼并转化为一张张独立的"便签"。这些便签是结构化数据的雏形，供下一步使用。

### 白板模式 (Whiteboard Mode)

这是整个工作流的中枢，模拟人在思考时"把东西摊在桌面上摆弄"的过程。

**定位**：结构化思考与逻辑梳理
**核心交互**：摆（拖拽与连接）
**用户行为**：将从"对话模式"中产生的"便签"拖入白板。用户通过移动、分组、连线等操作，探索这些便签之间的逻辑关系（比如因果、并列、包含）。
**AI角色**：一个逻辑分析师，观察用户的摆放方式，识别出信息的层级和结构。
**产出物**：备忘录 / 大纲。当用户在白板上完成了信息的布局，AI 会根据这种空间布局，自动生成一份结构清晰、逻辑连贯的"备忘录"或"大纲"。这标志着思路已经从"碎片化"变成了"系统化"。

### 画布模式 (Canvas Mode)

这是整个工作流的终点，模拟一个"演播室"或"指挥台"，用于最终的呈现和分发。

**定位**：策略性沟通与成果输出
**核心交互**：排（排序与配置）
**用户行为**：将"白板模式"生成的"备忘录"导入画布。在这里，摆放的位置不再仅仅代表逻辑关系，更代表"沟通的顺序"和"视觉的权重"。用户可以调整段落顺序，配置沟通对象和目的。
**AI角色**：一个沟通策略顾问和翻译官，根据用户的配置，对备忘录进行润色，将其转化为适合特定对象和场景的沟通文案。
**产出物**：沟通结果 / 工作流动作。最终的结果不是留在软件里的一张图，而是"发出去"的动作。它可以是一封格式完美的邮件、一条发给同事的钉/飞消息、一个创建好的任务单，或者一个生成的分享链接。


### 交互流程

1. 对话模式：灵感捕捉
2. 白板模式：结构化思考
3. 画布模式：策略性沟通
4. 对话模式：外部反馈
