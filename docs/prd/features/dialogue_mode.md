# 对话模式 (Dialogue Mode)

## 定位

灵感捕捉与信息入口

## 核心交互

聊（输入消息）

## 详细描述

这是整个工作流的起点，模拟人最自然的思维方式——"自言自语"或"与AI对话"。

### 用户行为

像使用聊天软件或大模型官网一样，通过一问一答的形式，把脑子里模糊的想法、遇到的问题、零散的信息"倒"出来。

### AI角色

一个聪明的倾听者和记录员，负责理解用户的意图，并将大段的对话内容切片。

### 产出物

便签。AI 会自动将对话中的关键信息、待办事项、灵感要点，提炼并转化为一张张独立的"便签"。这些便签是结构化数据的雏形，供下一步使用。

## 设计要点

1. **自然对话体验**
   - 支持多轮对话
   - 保持上下文连贯
   - 智能理解用户意图

2. **便签提取机制**
   - 自动识别关键信息
   - 支持用户手动标记
   - 便签可追溯来源对话

3. **快速采纳流程**
   - 一键采纳建议便签
   - 支持批量操作
   - 可编辑和完善便签内容
