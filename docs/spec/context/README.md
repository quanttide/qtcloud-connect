# 语境

🧠 沟通语境 = 认知语境（Cognitive Context）

## 定义

为支持人机协同解决尚未澄清、无法结构化、需持续探索的深层问题而构建的、可被人类理解且机器可操作的最小共识环境。

## 建模思想

消息、便签、笔记，这些都只是“数据”或“素材”，它们是语境的载体，但不是语境本身。语境是这些载体背后的关系、逻辑和意图网络。

我们可以把这个“语境结构”想象成一座城市的基础设施：
- 消息：街道上流动的车流（动态、实时、嘈杂）；
- 便签：路边临时的告示牌（提醒、待办、碎片）；
- 笔记：建筑物的砖瓦（静态、知识、沉淀）；
- 语境：城市的路网规划、水电系统和交通信号灯（结构、连接、规则）。
 
如果系统只是存储了车流、告示牌和砖瓦，那它只是一个“仓库”（甚至是“垃圾场”）。只有当系统理解了路网和水电（即语境结构），它才能被称为一个“活的系统”。

## 核心特征

### 问题导向

针对价值性混乱（如“如何商业化？”“方向在哪？”），而非已澄清的任务；
问题本身无明确答案，需通过持续对话逼近。

### 人机共构

人类提供直觉、意图与判断；
AI 提取结构、模式与潜在假设；
双方在迭代中校准对语境的理解。

### 动态演进

语境非一次性产出，而是随对话不断精炼、修正、分形；
每次交互都可能触发语境更新。

### 双模表达

图谱形式（供机器处理）：记录实体、关系、约束、冲突点；
文本摘要（供人理解）：高度凝练的问题边界、关键张力、当前共识。
